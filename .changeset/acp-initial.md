Add `@moonshot-ai/acp-adapter` package and `kimi acp` subcommand. Exposes the kimi-code agent over the Agent Client Protocol (ACP) so IDEs like Zed and JetBrains can drive a kimi-code session over stdio. Supports prompt streaming, tool calls, approvals, file reverse-RPC (`fs.read_text_file` / `fs.write_text_file`), multi-session control (`session/load` / `list` / `set_model` / `set_mode`), image and embedded-resource prompt content, MCP server forwarding from the client (HTTP transport only), and graceful SIGINT / SIGTERM shutdown. See `docs/zh/guides/ides.md` for IDE setup and `docs/zh/reference/kimi-acp.md` for the full capability matrix.

**Breaking (pre-release):** Phase 14 (PLAN D11) removes `NewSessionResponse.modes` and `LoadSessionResponse.modes`; both now publish `configOptions: SessionConfigOption[]` (the spec's unified picker surface, carrying model + mode side by side). The notification discriminator `current_mode_update` was replaced by `config_option_update`. A new `session/set_config_option` handler is the recommended way for clients to flip mode/model; the legacy `session/set_session_mode` and `session/unstable_set_session_model` paths still work for compatibility. This is pre-release breaking only — release notes will call it out at publish time.

**Breaking (pre-release):** Phase 15 splits thinking out of the model id into its own `SessionConfigBoolean` toggle (`id: 'thinking'`, `category: 'thought_level'`). The model dropdown no longer renders `,thinking` variant rows — each catalog entry surfaces exactly one option. The toggle appears dynamically based on the currently-selected model's `thinkingSupported` flag (omitted from the `configOptions` snapshot when not supported). The ACP `session/unstable_set_session_model({modelId:'...,thinking'})` legacy form is preserved: the adapter splits the suffix into a bare `Session.setModel` plus an implicit `Session.setThinking('high')`. Thinking is also now actually plumbed through to the SDK (`Session.setThinking`) on toggle change — previously the `,thinking` suffix was stripped and dropped, so the ACP-side thinking selection had no downstream effect.
